02 / 09
Science Layer · Implementation

Как работает привязка к научным данным

Процесс от выбора звезды из научного каталога до готовой NFT с верифицированными данными.

01
Импорт научных каталогов

Загружаем данные Hipparcos (118k звезд), Gaia DR3 (1.8 млрд), TESS (350k) и NASA Exoplanet Archive в PostgreSQL. Каждая звезда получает уникальный science_id и ссылку на оригинальный каталог.

CREATE TABLE science_catalog_entries ( id SERIAL PRIMARY KEY, catalog_id VARCHAR(50) NOT NULL, star_name VARCHAR(255), ra DOUBLE PRECISION NOT NULL, -- прямое восхождение J2000 dec DOUBLE PRECISION NOT NULL, -- склонение J2000 magnitude REAL, spectral_type VARCHAR(10), distance_ly REAL, exoplanet_count INTEGER DEFAULT 0, catalog_source VARCHAR(50), -- hipparcos | gaia | tess | nasa_exoplanet raw_data JSONB ); CREATE INDEX idx_catalog_sky ON science_catalog_entries (ra, dec);
02
Генерация NFT-звезд из каталога

Админ или алгоритм выбирает N звезд из каталога для очередного сектора ParagonSpace. Каждая звезда получает: координаты (x/y/z → пересчет из ra/dec), имя, редкость (на основе magnitude и спектрального класса), цвет (из спектра).

// Алгоритм пересчета научных координат в 3D-карту function catalogToStarMap(raDeg: number, decDeg: number, distanceLy: number) { const raRad = raDeg * Math.PI / 180; const decRad = decDeg * Math.PI / 180; const scale = Math.min(distanceLy / 100, 1); // normalize return { x: Math.cos(decRad) * Math.cos(raRad) * scale * 100, y: Math.sin(decRad) * scale * 100, z: Math.cos(decRad) * Math.sin(raRad) * scale * 50, }; }
03
Минт NFT с научными метаданными

При минте NFT смарт-контракт (или сервер) записывает science_id звезды в метаданные. Владелец получает не просто токен, а ссылку на реальный научный объект. Метаданные включают: название, каталог, RA/DEC, спектр, расстояние, количество экзопланет, ссылку на NASA/ESA страницу.

{ "name": "ParagonSpace Star #42", "science": { "catalog": "Hipparcos", "hipparcos_id": 87937, "hd_id": 168723, "constellation": "Lyra", "ra": 282.47, "dec": 38.78, "spectral_type": "A0V", "magnitude": 3.3, "distance_ly": 25.3, "exoplanets": 0 }, "image": "ipfs://Qm.../star_42.png", "attributes": [ ... ] }
04
Интеграция с космическими миссиями

Периодический cron-сервис проверяет: какие звезды из каталога попадают в поле зрения JWST, над какими пролетает ISS, какие видны с Марса. Владельцы получают уведомления и ачивки. Для SpaceX — подписка на API запусков и добавление NFT в полезную нагрузку (цифровой сертификат).

Ключевой принцип: Научные данные не меняются после минта. NFT хранит снапшот данных на момент выпуска. Это гарантирует, что владелец всегда может доказать подлинность привязки к реальному объекту. Обновления (новые снимки JWST, пролеты ISS) добавляются как achievements, не изменяя оригинал.

03 / 09
GameFi Engine · Implementation

Как работают игровые механики

Два слоя: on-chain (редкость, уровень, PvP-результаты) и off-chain (ресурсы, экспедиции, энергия) для снижения газовых затрат.

📊 On-chain (Smart Contract)
  • Владение NFT (ERC-721)
  • Редкость звезды (immutable после минта)
  • Уровень (level) — только владелец может повысить
  • PvP-результаты (победы/поражения)
  • Привязка к science_id
// Solidity — Star NFT с игровыми данными contract ParagonStar is ERC721, ERC2981 { struct StarData { uint256 scienceId; // ссылка на каталог uint8 rarity; // 0=common … 5=mythic uint16 level; // 1-99 uint32 wins; // PvP победы uint32 losses; // PvP поражения } mapping(uint256 => StarData) public stars; mapping(uint256 => address) public lastBattleOpponent; function levelUp(uint256 tokenId) external { require(ownerOf(tokenId) == msg.sender); require(stars[tokenId].level < 99); // Проверка наличия ресурсов (off-chain оракул) stars[tokenId].level++; } }
⚡ Off-chain (PostgreSQL + API)
  • Энергия звезды (восстанавливается по времени)
  • Ресурсы (Star Dust, Energy Crystals и др.)
  • Экспедиции (запуск, прогресс, завершение)
  • Созвездия (группировка звезд одного владельца)
  • Научные ачивки (JWST, ISS, Mars)
// TypeScript — Серверный GameFi Engine class GameFiEngine { async startExpedition(starId: number, expeditionId: number) { const star = await db.select().from(paragonStars) .where(eq(paragonStars.id, starId)).limit(1); const expedition = EXPEDITIONS.find(e => e.id === expeditionId); if (star.energy < expedition.requiredEnergy) throw new Error('Недостаточно энергии'); await db.insert(expeditionsLog).values({ starId, expeditionId, startTime: new Date(), endTime: addHours(new Date(), expedition.durationHours), }); return { status: 'started', endsAt: ... }; } async claimExpedition(starId: number) { const log = await db.select().from(expeditionsLog) .where(and( eq(expeditionsLog.starId, starId), isNull(expeditionsLog.claimedAt) )).limit(1); if (!log || new Date() < log.endTime) throw new Error('Экспедиция еще не завершена'); // Начисление ресурсов await db.update(starResources) .set({ star_dust: sql\`star_dust + $\{rewards.star_dust}\` }) .where(eq(starResources.starId, starId)); return { rewards }; } }

Почему on-chain + off-chain? Полностью on-chain GameFi сделал бы каждое действие (добыча, экспедиция, ачивка) дорогим в газе. Off-chain слой хранит ресурсы и энергию, а on-chain — только финальные результаты (уровень, PvP, редкость). Это дает экономию газа в 100–1000 раз.

04 / 09
NFT · Smart Contract · Tokenomics

Смарт-контракт и экономика токенов

ERC-721 с расширенными метаданными, EIP-2981 роялти и ERC-2771 для газлесс-транзакций.

📜 Smart Contract Architecture
  • ERC-721 — базовый стандарт NFT
  • ERC-2981 — роялти (5% автору, 5% ParagonSpace)
  • ERC-2771 — газлесс-транзакции (пользователь платит в токенах)
  • Merkle Tree — whitelist для ранних минтов
  • TokenURI dynamic — метаданные обновляются при апгрейдах
💰 Tokenomics
// Структура доходов с минта одной звезды Primary Sale: $50–$500 (зависит от редкости) ├─ 70% → Разработка и операционные расходы ├─ 15% → Научное партнерство (NASA/ESA/университеты) ├─ 10% → Маркетинг и комьюнити └─ 5% → Команда и advisors Secondary Sale (перепродажа): 10% royalty ├─ 5% → ParagonSpace (развитие платформы) ├─ 2.5% → Научный фонд (гранты и данные) ├─ 1.5% → Вознаграждение владельцу звезды └─ 1% → DAO Treasury
🔗 Deployment Plan
  • Base (L2) — низкие комиссии, экосистема Coinbase
  • Polygon — альтернатива с большой NFT-аудиторией
  • Ethereum — основной L1 для премиум-минтов
  • Cross-chain bridge в будущем (LayerZero / CCIP)
🔐 Безопасность
  • Аудит смарт-контракта (Trail of Bits / OpenZeppelin)
  • OpenZeppelin Contracts — проверенные библиотеки
  • Timelock на upgradeable контракты (48ч задержка)
  • Multi-sig (Gnosis Safe) для административных функций

Зачем Base L2? Mint на Ethereum Mainnet может стоить $50–200 газа. На Base (Coinbase L2) средняя комиссия — $0.01–0.05. Это делает GameFi-механики (апгрейд, баттлы) экономически доступными. Для премиум-секторов оставляем возможность минта на Ethereum.

Science API · SpaceX · NASA · ESA

Интеграция с космическими API

Как ParagonSpace подключается к реальным данным из космоса.

🔭 NASA APIs
  • NASA Exoplanet Archive API — получение списка звезд с экзопланетами, обновление раз в неделю
  • MAST Portal API (STScI) — данные JWST, Hubble, TESS. Поиск изображений по координатам
  • NASA Image & Video Library — официальные изображения для NFT-метаданных
  • ISS Tracker API — реальное положение станции, расчет зенита над звездами
🚀 SpaceX & Commercial
  • SpaceX Launch API (unofficial) — данные о запусках, полезной нагрузке, окнах старта
  • Space-Track.org — TLE-данные спутников Starlink для трекинга
  • Blue Origin Launch Schedule — публичные данные о полетах New Shepard
  • Celestrak — орбитальные данные для ISS и тысяч спутников

  3. Если звезда получила новый JWST-снимок или над ней пролетела ISS — владельцу приходит уведомление
  4. Уведомление может быть: email, push, in-app или on-chain (через событие в контракте)
  5. Для SpaceX: подписка на RSS/JSON фид запусков + ручная верификация перед привязкой к NFT
